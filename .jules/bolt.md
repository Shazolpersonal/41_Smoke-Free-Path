## 2026-05-03 - Prevent 41 Unnecessary StepCard Re-renders
**Learning:** In list implementations (e.g. `ProgressCalendar` and `TrackerScreen`), passing inline arrow functions to memoized child components (`React.memo(StepCard)`) breaks shallow equality. Every parent re-render causes all 41 list items to re-render, creating a significant UI bottleneck for complex grids.
**Action:** Always extract inline list item handlers into stable `useCallback` functions at the parent level. Update child components to handle specific arguments internally (e.g., changing `onPress: () => void` to `onPress: (id: number) => void` and invoking `() => onPress(id)` within the child's `Pressable`). This allows `React.memo` to work as intended.
